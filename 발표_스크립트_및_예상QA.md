# 🎤 항공편 지연 예측 모델 학원 프로젝트 발표 대본

안녕하십니까.

저는 김종록이며, 이번 프로젝트는 AI 기반 항공편 지연 예측 모델 개발 및 시각화 시스템 구축입니다.

### 1. 프로젝트 개요 (1~3페이지)

최근 항공편 지연은 공항 운영 효율성과 고객 만족도에 큰 영향을 주고 있습니다.

이에 따라 항공 운항 데이터를 활용하여 항공편의 지연 여부를 사전에 예측하고, 운영자가 쉽게 활용할 수 있는 웹 기반 시스템을 구축하고자 본 프로젝트를 진행하였습니다.

데이터 수집부터 전처리, 모델 학습, 성능 평가, 그리고 Streamlit 기반 웹 서비스 구현까지 전체 AI 파이프라인을 구축하였습니다.

그럼 실제로 사용한 데이터에 대해 설명드리겠습니다.

### 2. 데이터 수집 및 구성 (4페이지)

본 프로젝트에서는 DACON의 항공편 지연 예측 데이터를 활용하였습니다.

전체 약 100만 건의 데이터 중 정답값이 존재하는 25만 건의 데이터를 학습에 사용하였습니다.

예측 대상인 Target 변수는 Delay이며,
0은 정상 운항,
1은 지연을 의미합니다.

주요 입력 변수로는
- 운항 월
- 출발 예정 시간
- 비행 거리
- 항공사
- 출발 공항
- 도착 공항

총 6개의 주요 변수를 활용하였습니다.

### 3. 데이터 전처리 (5~6페이지)

AI 모델의 성능 향상을 위해 데이터 전처리를 수행하였습니다.

먼저 정답값인 Delay가 없는 데이터는 제거하여 학습 데이터의 신뢰성을 확보하였습니다.

출발 예정 시간 변수에는 일부 결측치가 존재했는데, 해당 데이터를 삭제하면 데이터 손실이 발생하기 때문에 중앙값으로 대체하였습니다.

또한 예측에 직접적인 도움이 되지 않는 변수들은 제거하였습니다.
예를 들어 ID, 항공기 등록번호, 결항 여부 등의 변수는 모델 학습에서 제외하였습니다.

그리고 Feature Engineering을 통해 출발 예정 시간에서 시간 정보만 추출하여 Hour라는 새로운 변수를 생성하였습니다.
이를 통해 모델이 오전, 오후, 야간 등 시간대별 패턴을 더 효과적으로 학습할 수 있도록 하였습니다.

범주형 변수인 항공사와 공항 정보는 One-Hot Encoding을 적용하였고,
수치형 변수는 StandardScaler를 이용하여 표준화하였습니다.

### 4. 탐색적 데이터 분석 (6~7페이지)

다음은 EDA, 즉 탐색적 데이터 분석입니다.

먼저 변수 간 상관관계를 확인하기 위해 히트맵을 분석하였습니다.
분석 결과 시간과 거리 같은 단순 수치형 변수들 간의 상관관계는 매우 낮게 나타났습니다.
이는 항공편 지연이 단순한 변수 하나가 아닌 여러 변수의 복합적인 영향을 받는다는 것을 의미합니다.

또한 데이터 분포를 확인한 결과 출발 시간은 오전 6시에서 8시 구간에 가장 많이 집중되어 있었으며,
비행 거리는 약 500마일 전후의 단거리 노선이 가장 많은 것으로 나타났습니다.

### 5. 모델 학습 및 비교 (7~8페이지)

다음으로 모델 학습을 진행하였습니다.
이번 프로젝트에서는
- Logistic Regression
- Random Forest
- XGBoost
총 3개의 모델을 비교하였습니다.

비교 결과를 보시면,
Logistic Regression은 복잡한 패턴을 학습하는 데 한계가 있었고,
Random Forest는 정확도는 높았지만 지연 데이터를 잘 찾아내지 못하는 문제가 있었습니다.

반면 XGBoost는 ROC-AUC 0.785, Recall 65%로 가장 우수한 성능을 보여 최종 모델로 선정하였습니다.

**XGBoost를 선택한 이유**
여기서 중요한 점은 데이터 불균형 문제였습니다.
실제 항공편 데이터는 대부분 정상 운항이고 지연 데이터는 상대적으로 적습니다.
이 때문에 단순히 정확도만 보면 성능이 좋아 보일 수 있지만, 실제로는 지연 항공편을 놓치는 문제가 발생합니다.

그래서 XGBoost의 `scale_pos_weight` 옵션을 활용하여 지연 데이터에 더 높은 가중치를 부여하였습니다.
그 결과 Recall을 5%에서 65%까지 향상시킬 수 있었습니다.

### 6. 변수 중요도 분석 (8페이지)

다음은 Explainable AI 기반 변수 중요도 분석입니다.

모델이 왜 특정 항공편을 지연이라고 판단했는지 분석하기 위해 Feature Importance를 확인하였습니다.
분석 결과 단순히 거리나 시간보다 특정 항공사와 특정 공항이 지연 여부에 더 큰 영향을 미치는 것으로 나타났습니다.
이를 통해 모델의 예측 근거를 사용자에게 설명할 수 있도록 하였습니다.

### 7. Streamlit 웹 시스템 구현 (9~11페이지)

다음은 실제 구현한 웹 시스템입니다.
Streamlit을 활용하여 사용자가 직접 조건을 입력하고 결과를 확인할 수 있는 대시보드를 개발하였습니다.

사용자는 항공사, 출발 공항, 도착 공항, 운항 월, 비행 거리, 출발 시간 등을 입력하면 실시간으로 지연 확률을 확인할 수 있습니다.

또한 화면은
- 실시간 예측 결과
- 원인 분석
- 데이터 및 모델 정보
총 3개의 탭으로 구성하였습니다.

특히 XAI 기능을 적용하여 예측 결과뿐 아니라 어떤 요인이 지연에 가장 큰 영향을 미쳤는지도 그래프로 시각화하였습니다.

*(여기서 실제 시연하면서 설명하면 좋습니다.)*
예를 들어 출발 공항과 항공사를 변경하면 지연 확률이 실시간으로 바뀌고, 아래 그래프에서 어떤 요소가 가장 큰 영향을 미쳤는지 확인할 수 있습니다.

### 8. 한계점 및 개선 방향 (12페이지)

마지막으로 한계점입니다.

현재 모델에는 기상 데이터가 포함되어 있지 않습니다.
하지만 실제 항공편 지연에는 강수량, 풍속, 태풍 등의 날씨 요인이 큰 영향을 미칩니다.
따라서 향후에는 기상청 API와 연동하여 기상 데이터를 추가할 계획입니다.

또한 데이터 불균형 문제를 더욱 개선하기 위해 SMOTE와 같은 오버샘플링 기법과 앙상블 기법을 적용할 예정입니다.
