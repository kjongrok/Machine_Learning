# ✈️ 항공편 지연 예측 프로젝트 진행 현황 (PROGRESS)

## 📌 완료된 작업 내역 (오늘의 성과)

### 1. 데이터 분석 및 AI 모델링 고도화
- DACON 항공편 운항 원시 데이터(약 100만 건) 중 정답셋 25만 건 추출 및 전처리 파이프라인 구축 완료
- **데이터 불균형(Class Imbalance) 문제 해결**
  - 초기 Random Forest 모델의 **Accuracy Paradox(정확도는 82%이나, 실제 지연은 5%만 감지)** 현상 돌파
  - **XGBoost** 알고리즘 도입 및 `scale_pos_weight=5` 하이퍼파라미터 가중치 튜닝 적용
  - 결과적으로 지연 비행기 선제 탐지율(**Recall**)을 기존 5%에서 **65%로 수직 상승**시키는 실무 맞춤형 고성능 모델 완성
- Scikit-learn `Pipeline`을 활용하여 전처리(StandardScaler, OneHotEncoder) 및 모델(XGBoost)을 하나의 모델 파일(`flight_pipeline.pkl`)로 패키징 성공

### 2. 웹 대시보드 (Streamlit) 프로토타이핑 및 UI/UX 고도화
- `app.py`를 통해 사용자 친화적인 와이드 레이아웃 대시보드 개발 및 **조건 선택창 한글화(공항/항공사명 매핑)** 완벽 적용
- 가독성을 극대화하기 위해 메인 화면을 **3단 탭(Tabs)** 구조(예측 결과 / AI 원인 분석 / 모델 신뢰도 정보)로 전면 개편
- **국소적 해석(Local Explanation) 기반 XAI(설명 가능한 AI) 고도화**
  - 전체 학습 데이터 기준(Global)이 아닌, **'사용자가 현재 선택한 비행 조건 6가지' 내에서만 필터링**하여 특정 비행편 지연의 진짜 원인을 찾아내는 동적 막대그래프 구현
  - `Month(운항 월)` 등 직관적인 포맷의 동적 텍스트 해석 알림창을 추가하여, 비전문가도 쉽게 AI의 판단 근거를 이해할 수 있도록 설계

### 3. 최종 산출물(실무 보고서) 작성 및 작업 환경 정비
- 사용자(대표님)가 제공한 공공기관 용역과제 11페이지 폼(양식)의 목차와 표 구조를 100% 반영하여 풀버전 HTML 보고서(`머신러닝_산출물_최종_보고서.html`) 제작 완료
- 보고서에 들어갈 필수 시각화 이미지 5종(Heatmap, Histogram, 튜닝 전후 Confusion Matrix 비교, Feature Importance)을 파이썬 스크립트로 직접 생성 및 삽입
- 프로젝트 폴더 내 불필요했던 임시 데이터 추출 스크립트 및 구버전 파일 등 총 9종 삭제 및 환경 청소 완료

### 4. 보고서 퀄리티 향상 (머신러닝 스토리텔링 고도화)
- 기존 학습 예제(`ml.ipynb`, `ml2.ipynb`)에서 다루었던 원핫 인코딩, StandardScaler, SimpleImputer 전처리 파이프라인과 GridSearchCV 하이퍼파라미터 튜닝 기법을 도입한 과정을 전문적인 스토리텔링 형식으로 보고서에 추가
- 분류 문제에 적합한 평가지표(ROC-AUC, Recall, Accuracy)를 기준으로 한 모델 비교(Logistic Regression vs Random Forest vs XGBoost) 표를 삽입하여 최종 모델 선정의 학술적 타당성 대폭 강화
