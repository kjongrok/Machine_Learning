# ✈️ 항공편 지연 예측 프로젝트 진행 현황 (PROGRESS)

## 📌 완료된 작업 내역 (최종 성과)

### 1. 데이터 분석 및 AI 모델링 고도화
- DACON 항공편 운항 원시 데이터(약 100만 건) 중 정답셋 25만 건 추출 및 전처리 파이프라인 구축 완료
- **데이터 불균형(Class Imbalance) 문제 해결**
  - 초기 Random Forest 모델의 **Accuracy Paradox(정확도는 82%이나, 실제 지연은 5%만 감지)** 현상 돌파
  - **XGBoost** 알고리즘 도입 및 `scale_pos_weight=5` 하이퍼파라미터 가중치 튜닝 적용
  - 결과적으로 지연 비행기 선제 탐지율(**Recall**)을 기존 5%에서 **65%로 수직 상승**시키는 실무 맞춤형 고성능 모델 완성
- Scikit-learn `Pipeline`을 활용하여 전처리(StandardScaler, OneHotEncoder) 및 모델(XGBoost)을 하나의 모델 파일(`flight_pipeline.pkl`)로 패키징 성공

### 2. 웹 대시보드 (Streamlit) 프로토타이핑 및 UI/UX 고도화

- 가독성을 극대화하기 위해 메인 화면을 **3단 탭(Tabs)** 구조(예측 결과 / AI 원인 분석 / 데이터 및 모델 정보)로 전면 개편

## 🚨 트러블슈팅 (오늘 직면했던 문제점 및 해결 과정)

### 1. SHAP(설명가능한 AI) 라이브러리 충돌 및 시각적 한계
- **문제 발생:** XGBoost 최신 버전과 `shap.TreeExplainer` 간의 `base_score` 메타데이터 파싱 에러(ValueError)가 발생함. 이를 `PermutationExplainer`로 우회하여 해결했으나, 원핫 인코딩된 범주형 변수(공항, 항공사 등)가 너무 많아 SHAP 점도표(Summary Plot)가 시각적으로 매우 빈약하고 난해하게 출력됨.
- **해결 방안:** 무리하게 SHAP을 고집하여 보고서 퀄리티를 떨어뜨리기보다는, 심사위원이 직관적으로 이해하기 쉬운 전통적인 **변수 중요도(Feature Importance) 막대그래프**로 롤백하고 깔끔하게 시각화하여 보고서 전문성을 방어함.

### 2. 한글(HWP) 프로그램 튕김 및 레이아웃 붕괴 문제
- **문제 발생:** 사용자가 기존 양식(02폴더 수질 예측 템플릿)에 HTML 보고서 내용을 수동으로 복사-붙여넣기 하던 중, 5페이지 부근에서 한글 프로그램이 뻗어버림(Crash). 또한 일반적인 변환 스크립트로는 표 크기와 글씨 간격이 뒤죽박죽으로 깨지는 현상 발생.
- **해결 방안:** 파이썬의 `python-docx` 라이브러리를 활용하여 단순 텍스트 변환이 아닌 **'양식 복제기(Formatter)' 수준의 세밀한 스크립트(`generate_docx3.py`)를 직접 코딩**함.
  - 여백 상하좌우 1.0인치, 줄간격 1.15, 문단 여백 0, 폰트 맑은 고딕(11pt/10pt 분리), 표 열 비율 등을 하드코딩하여 수작업을 완벽히 대체한 최종 제출용 문서(`머신러닝_산출물_최종_보고서_제출용(김종록).docx`)를 무사히 자동 생성해냄.
